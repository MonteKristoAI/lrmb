# SDS / Warrior — Banned Vocabulary

**Source:** Christopher Knight's written feedback on 2026-04-01 and 2026-04-07. These are not stylistic preferences. Each was personally flagged by the final copy approver at SDS. Reintroducing them is a credibility hit with the client.

---

## Hard-Banned Terms

### "Interoperable" / "Interoperability"
- **Why:** "Too complicated — just a tad bit too mouthful." Oilfield buyers won't parse it.
- **Use instead:** **Universally Compatible** / **Universal Compatibility**
- Global find-and-replace across every page.

### "Subsurface Intelligence"
- **Why:** "Sounds like a buzzword that will not land with most of our customers."
- **Use instead:** "Wireline Acquisition" / "Downhole Data Acquisition" / "Mission-Critical Wireline Acquisition and Control"
- Note: Christopher allowed that a future cloud visualization product could use this term — not for the current launch.

### "Data Sovereignty" / "Switzerland of Data" / "Your data is yours"
- **Why:** "Not a selling point, more like a basic requirement of a third-party system. Everyone already prioritizes ease of access." Doesn't differentiate Warrior from anything.
- **Use instead:** Emphasize **ease of data access** — data streaming, many export options, easy manipulation. That's the real selling point.
- Drop the "sovereignty" framing entirely.

### "Guided Workflows" / "Guided Automation" / "Intelligent Automation" / "Supervised Automation"
- **Why:** Warrior does NOT automate setup. The only automation in the product is **Auto-Perf mode** (firing sequence for perforating).
- **Use instead:** **Standardized Interfaces** / **Purpose-Built Interfaces**
- Rationale: Warrior's value is a consistent interface regardless of tool string — not guidance. "Standardized" describes the architecture truthfully; "guided" implies a feature that doesn't exist.

### "Proprietary formats"
- **Why:** Too abstract. Doesn't describe the pain the operator actually feels.
- **Use instead:** **Closed-system limitations** / **Black-box ecosystems** / **Closed ecosystems**

### "Old" / "Legacy"
- **Why:** Negative framing. Violates the "Modern Wisdom" identity pillar.
- **Use instead:** **Field-Validated** / **Field-Proven** / **Battle-Tested**

### "Workflows" (in the context of perforating or logging)
- **Why:** Christopher flagged: "Should be 'standardized interfaces,' not workflows. Workflows implies guidance we don't provide."
- **Use instead:** **Interfaces** / **Standardized interfaces**

### "Tractor-conveyed" / "High-power direct-connect" claims
- **Why:** These capabilities are planned but **not shipping**. Cannot imply features Warrior doesn't have.
- **Use instead:** Nothing. Drop entirely.

## Product Naming Rules

### Warrior vs Warrior 8 vs Warrior Universal Panel
- The **launch** is the **Warrior Universal Panel** (ULPP). Hardware.
- **Warrior 8** = existing software line (internally P10V7.1 — P10 is Warrior 8, V7 is the 7th major version, .1 is the quarterly release). Already in the market for years. **Not launching.**
- When talking about the platform generally, use **Warrior**.
- Never write "Warrior 8.0 launch" or "launching Warrior 8" — both are wrong.

### Warrior Universal Panel specifics
- Christopher: "When talking to ETs about the new panel, say 'the Warrior Universal Panel' specifically — they already own Warrior, we're selling them the new panel."
- In ET / Tech Manager copy, replace generic "Warrior" references with "Warrior Universal Panel" where the new hardware is the subject.

## Persona-Specific Guidance (from Field Messaging Guide alignment)

### Field Logging Engineer
- "Off the reel" → "When a log is given to the customer"
- Don't use "Windows OS" as a proof point — everyone runs Windows now.

### Perforating Engineer ("The Shooter")
- Don't say "clean data" in a perforating context. Perf delivers **"consistent, well-executed stages that keep Frac operating on schedule."**
- Focus on: running every major gun system and addressable switch from one panel without reconfiguration.
- Don't frame "my guys know the job" as an objection — not a real pushback. Real competitors are **Dyna SPS, Titan ControlFire, G&H DSS.** Address those.

### Field Service Manager
- FSM copy should focus on **perforating crews running 24/7**, not CH/OH logging.
- Don't claim "job templates pushed across crews" — Warrior supports **standardized interface configurations**, not templates.
- Don't promise "how-to guides and video library" — they're outdated. Emphasize **diagnostic and troubleshooting capabilities** instead.

### Electronics Technician / Tech Manager
- Refocus on **Perforating ET / Technology Manager** (ETs don't attend shows; the decision-maker is the Tech Manager who used to be an ET).
- Hook: "When a panel, a CCL, or even a switch comes back from the field with a reported failure — how often is the fault actually in the shop vs. testing fine and going back out?"
- Don't compare Warrior to legacy systems — customers already switched years ago. Compare the **Universal Panel** to their current Warrior setup.

### Tool Technician
- Soften any claims about documentation being current.
- Emphasize the **user forum** as the real support layer.

## Competitive Landscape (mention with care)

These are the real competitors Christopher named:
- **SLB, Halliburton, Weatherford** — the "proprietary black-box" majors (closed systems). Frame as the problem Warrior solves.
- **Dyna SPS, Titan ControlFire, G&H DSS** — perforating-specific competitors. Don't dismiss them. Position Warrior as the system that works when the client **changes the switch** (their weakness: each locks to one switch type).

## Approved Reframings (the "do say" list)

| Don't say | Do say |
|---|---|
| Interoperable with every tool | Universally compatible with every major wireline tool manufacturer |
| Subsurface Intelligence | Wireline Acquisition / Mission-Critical Wireline Acquisition and Control |
| Data Sovereignty | Easy data access · many export options · real-time streaming |
| Guided workflows | Standardized interfaces · purpose-built interfaces |
| Legacy / old systems | Field-validated · field-proven · battle-tested |
| Proprietary formats | Closed-system limitations · black-box ecosystems |
| Warrior 8.0 launch | Warrior Universal Panel launch |
| "Your data is yours" | "Easy to access. Easy to move. Easy to share." |
| Vendor lock-in | Open interoperability (or: universal compatibility) |
| Important | Mission-Critical |
| Compatible | Universally compatible |
| Well logging software | Wireline acquisition platform |

---

**Rule of thumb:** If you're about to write a term that sounds "techy" or "marketingy," stop. The audience is wireline field engineers in Houston and the Permian. They can smell corporate speak from a mile away. Write like you've been on location. The manifesto's phrase is: **"We speak your language because we've walked your site."**
