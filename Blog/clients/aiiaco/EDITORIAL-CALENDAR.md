## Buffer week protocol

Buffer weeks (6, 15, 24) are for:
1. Refresh sprint on top 3 posts losing position
2. Rapid-response posts when regulatory news breaks (new CFPB rule, EU AI Act guidance update, etc.)
3. Case study publish if Nemr closes a notable engagement
4. Nothing (if the sprint is on-track and no news) - take the week, no forced content

## Quality gates per week

Before publishing:

- [ ] Post passes `content-quality-gate.md` (0 banned words, 0 em/en dashes)
- [ ] Post has Direct Answer block (40-80 words)
- [ ] Post has 5+ FAQ pairs
- [ ] Post cross-links to at least 1 aiiaco.com service page
- [ ] Post has at least 3 named entities from the 30-entity graph
- [ ] Schema validates via `curl /{slug}/ | grep application/ld+json`
- [ ] Template rotation obeyed (check last 2 published posts)

After publishing:

- [ ] `wrangler pages deploy dist --project-name=aiiaco-blog` success
- [ ] All URLs return HTTP 200 (index, post, sitemap, rss)
- [ ] Git push to `MonteKristoAI/aiiaco-blog` main
- [ ] `SITEMAP.md` updated with new row
- [ ] Citation check scheduled for Friday of the same week

## Weekly workflow

Monday 09:00 EST:
1. Finalize and publish this week's post
2. Update `SITEMAP.md` with publish row

Monday 17:00 EST:
3. Begin drafting next week's post outline

Wednesday:
4. First draft of next week's post complete
5. Content-quality-gate self-check

Friday 10:00 EST:
6. Weekly citation check on priority 10 posts (Google AI Overview, Perplexity, ChatGPT Web Search)
7. Log results in `SITEMAP.md` citation tracking table
8. Final pass on next week's post

Friday 17:00 EST:
9. Schedule next week's post for Monday publish
